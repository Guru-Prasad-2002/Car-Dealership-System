Introduction

The Car Dealership project is a comprehensive system designed to assist the owner of a car showroom chain in managing various aspects of their business. The system includes key features such as the ability to enter and maintain details about showrooms, car models, and employees, delete unwanted data as needed, record and track car sales transactions, restock car models as needed, update information about showrooms, cars, and employees, and display a range of business-related details, such as the showrooms in the chain, the car models available, sales made, and known customers. The overall goal of the Car Dealership project is to provide a streamlined and efficient method for the owner to manage their business, stay on top of important details, and transactions. This project aims to study the various features of the system and evaluate its effectiveness in achieving the desired goals.

Description of Various Attributes

Car

The Car table has the following attributes:
●car_id: an integer that serves as the primary key and unique identifier for each car in the table.
●model: a string that stores the model of the car.
●year: an integer that stores the year of the car.
●color: a string that stores the color of the car.
●price: an integer that stores the price of the car.
●quantity: an integer that stores the quantity of that particular car model available.

Customer

The Customer table has the following attributes:
●customer_id: an integer that serves as the primary key and unique identifier for each customer in the table.
●name: a string that stores the name of the customer.
●address: a string that stores the address of the customer.
●phone: an integer that stores the phone number of the customer.

Showroom

The Showroom table has the following attributes:
●showroom_id: an integer that serves as the primary key and unique identifier for each showroom in the table.
●address: a string that stores the address of the showroom.
●manager_id: an integer that stores the id of the manager of the showroom.

Manager

The Manager table has the following attributes:
●manager_id: an integer that serves as the primary key and unique identifier for each manager in the table.
●name: a string that stores the name of the manager.
●salary: an integer that stores the salary of the manager.
●showroom_id: an integer that stores the id of the showroom where the manager is working.

Sales

The Sales table has the following attributes:
●sales_id: an integer that serves as the primary key and unique identifier for each sale in the table.
●car_id: an integer that stores the id of the car that was sold.
●customer_id: an integer that stores the id of the customer who made the purchase.
●showroom_id: an integer that stores the id of the showroom where the car was sold.
●date: a date-time object that stores the date and time of the sale.
●price: an integer that stores the price of the car at the time of the sale.

Table Relationships

The relationships between the tables are:
●Car table has one to many relationship with Sales table (1 car can be sold many times)
●Customer table has one to many relationship with Sales table (1 customer can buy many cars)
●Showroom table has one to many relationship with Sales table (1 showroom can sell many cars)
●Showroom table has one to one relationship with Manager table (1 showroom has 1 manager)
●Manager table has one to one relationship with Showroom table (1 manager manages 1 showroom)

Queries

The home page of our system is designed to provide the user with various views on the data stored in the database. In order to achieve this, the page implements the following four queries:
●Managers earning above a certain threshold: This query retrieves the managers from the "manager" table whose salary is above the specified threshold. This allows the user to see which managers are earning the most and make informed decisions.
●Most expensive and least expensive car models: This query retrieves the car models from the "car" table and orders them by price, displaying the most and least expensive models. This allows the user to see which car models are the most profitable and which ones are not selling well.
●Most regular customers: This query retrieves the customers from the "customer" table and orders them by the number of purchases they have made. This allows the user to identify which customers are the most loyal and make informed decisions on how to retain them.
●Car models with highest and least sales: This query retrieves the car models from the "car" table and orders them by the number of sales they have. This allows the user to identify which car models are the most popular and make informed decisions on which models to stock more of.

Sell

The sales page of our system is designed to facilitate the process of making a sale by allowing the admin to enter the car ID and the sales ID. On submitting the form, the corresponding action is performed in the backend, updating the relevant tables in the database. Specifically, when a sale is made, the number of cars in the inventory for that particular car ID is reduced by one. Additionally, a new entry is made in the sales table, recording the details of the sale such as car ID, customer ID, and showroom ID. Overall, the sales page provides an easy and efficient way to make a sale, allowing the user to quickly and easily update the inventory and sales records in the database. This improves the overall workflow of the system and helps the user to keep track of the sales easily.
